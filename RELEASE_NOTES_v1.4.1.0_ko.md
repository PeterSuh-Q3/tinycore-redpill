# alpine-redpill v1.4.1.0

`alpine-redpill` 라인의 후속 수정 릴리즈입니다. 저사양 메모리 장비에서 Alpine 빌드 환경을 실제로 쓸 수 있게 만드는 것과, v1.4.0.0 이후 발견된 자잘한 문제들을 정리하는 데 집중했습니다.

<details>
<summary><b>저사양 RAM(2GB) 빌드 환경에서 더 이상 메모리 부족이 발생하지 않습니다 (클릭해서 펼치기)</b></summary>

<br>

이번 릴리즈에서 가장 중요한 수정입니다. 빌드 환경에 2GB RAM이 할당된 시스템에서 로더 빌드 과정(`redpill-load`)이 진행 도중 메모리가 고갈되어 커널 OOM(Out Of Memory) killer에 의해 조용히 강제종료되는 문제가 있었습니다 — GUI 세션이 통째로 죽거나, 패키지 설치 단계에서 "No space left on device" 오류로 나타나기도 했습니다.

**원인**: Alpine의 디스크리스 루트 파일시스템은 tmpfs(RAM 기반)이고, 커널 기본값은 총 RAM의 50%까지만 허용합니다. 반면 TinyCore는 루트 tmpfs를 RAM의 약 90%까지 쓸 수 있었고, 게다가 자체적으로 압축된 읽기전용 확장 마운트 방식을 써서, 같은 하드웨어에서도 훨씬 넓은 실질 작업 공간을 확보하고 있었습니다.

**수정 내용** (총 RAM이 2.1GB 이하일 때만 자동 적용):
- 루트 tmpfs 상한을 커널 기본값(50%)에서 **90%**로 확대 — TinyCore와 동등한 수준
- 예비로 비워두던 3번 파티션에 **1GB 디스크 기반 스왑파일**을 생성 — zram 같은 RAM 기반 방식은 먼저 시도해봤지만 실제 메모리 압박 상황에서는 부족함을 확인했고, 물리 RAM을 초과하는 진짜 디스크 여유 공간을 확보하는 방식으로 최종 결정

DSM 로더 전체 빌드(SA6400 플랫폼)를 2GB RAM에서 무인으로 완주시켜 검증했습니다 — 수정 전에는 매번 실패했고, 수정 후에는 OOM 발생 없이 정상 완료됩니다.

<details>
<summary><b>그 외 수정 사항 (클릭해서 펼치기)</b></summary>

<br>

- **메뉴에서 ESC/Cancel을 눌러도 종료되지 않음.** 메인 mshell 메뉴에서 실수로 Esc나 Cancel을 누르면 바로 셸로 빠져나갔습니다. 이제는 메뉴가 그대로 다시 표시되며, 종료는 메뉴의 명시적인 "Exit" 항목을 선택했을 때만 가능합니다.
- **GRUB 배경 이미지 경로 수정.** 로더 빌드 부팅 메뉴의 배경 이미지가 (이제는 비어있는) 3번 파티션을 찾고 있어서, 실제로는 4번(`alpine`) 파티션에 있는데도 배경이 조용히 누락되던 문제를 수정했습니다.
- **미사용 죽은 코드 제거.** fdisk 경로 해석 로직에서 정의되지 않은 변수를 참조하던, 실행될 일 없는 분기를 정리했습니다.

</details>

## 참고

- 기존 v1.3.1.1(TinyCore), v1.4.0.0 사용자는 직접 업데이트하지 않는 한 영향받지 않습니다.
- RAM/스왑 튜닝은 RAM이 2.1GB 이하인 시스템에서만 활성화되며, 그 이상인 시스템은 그대로입니다.
