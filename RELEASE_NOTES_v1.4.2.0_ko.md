# alpine-redpill v1.4.2.0

안정화 마일스톤 릴리즈입니다. mshell/xTCRP 양쪽에 걸친 다수의 구조적 수정과 아키텍처 개선이 누적되어, `1.4.1.x` 패치 릴리즈보다 더 큰 버전 점프로 배포합니다.

<details>
<summary><b>DSM .pat 파일이 대폭 경량화됩니다 (클릭해서 펼치기)</b></summary>

<br>

Synology `.pat` 파일은 보통 300~400MB인데, 대부분 DSM OS 자체 설치용 페이로드(`hda1.tgz`, 번들 `.spk` 패키지 등)로 로더 빌드 과정에서는 전혀 쓰이지 않습니다. `redpill-load`의 전체 플랫폼 config를 실측 스캔한 결과, 로더 빌드가 실제 `.pat`에서 필요로 하는 파일은 단 5개(`zImage`, `rd.gz`, `GRUB_VER`, `grub_cksum.syno`, `VERSION`, 합쳐서 약 9MB)뿐임을 확인했습니다.

이번 릴리즈는 이 발견을 두 지점에 적용했습니다:
- **영구 캐시 사본**(같은 모델/리비전 재빌드 시 재사용되는 디스크 캐시)이 원본 전체 대신 이 5개 파일만으로 재압축됩니다 — 모델당 **약 97% 절감**(~400MB → ~10MB).
- **빌드 자체**도 `.pat` 전체를 tmpfs에 풀어놓는 대신 이 5개만 선택적으로 추출해, 저사양 환경에서 빌드 중 OOM을 유발하던 가장 큰 원인을 직접적으로 줄였습니다.

두 경우 모두 5개 파일 중 하나라도 아카이브에서 실제로 확인되지 않으면(가정이 아니라 실제 목록 대조) 기존 전체 추출 방식으로 안전하게 폴백하므로, 특이한 플랫폼/버전 조합에서 조용히 깨진 로더가 만들어질 위험이 없습니다.

</details>

<details>
<summary><b>그 외 안정화 수정 (전부 실기에서 재현·수정 확인, 클릭해서 펼치기)</b></summary>

<br>

- **curl `-n`(`--netrc`) 오타**: `redpill-load`의 확장자 다운로더가 이 플래그 때문에 `~/.netrc`가 없는 모든 박스에서 확장자 다운로드(예: `eudev`)가 **매번** 실패하고 있었습니다. 간헐적 네트워크 문제가 아니라 구조적 버그였고, v1.4.1.2에서 추가한 다운로드 진단 로그 덕분에 발견됐습니다.
- **`/lib64` 심볼릭 링크 체크**: `gcompat`이 이미 `/lib64`를 실제 디렉토리로 만들어두는 환경에서 잘못된 검사(`/lib64` 자체에 대한 `-h`/`-d`)를 쓰고 있어, 재빌드할 때마다 `ln: File exists` 오류가 났습니다.
- **xTCRP SSH 세션에서 다이얼로그 메뉴가 조용히 멈추는 문제**: 이 환경의 sshd가 SSH를 통한 `TERM` 협상을 제대로 안 해서 `TERM=dumb`으로 떨어지는데, 이 상태에선 `dialog`가 초기화에 실패하고 메뉴 루프가 화면 표시 없이 무한 재시도합니다. `menu.sh`가 이제 `TERM`이 없거나 `dumb`이면 `linux`로 보정합니다.
- **xTCRP가 1년 넘게 방치된 `main` 브랜치로 회귀하는 문제** — 이번에 두 곳 더 발견/수정: `xtcrp.tgz` 복원 URL, 그리고 `functions.sh` 자신의 `$build` 변수(둘 다 `is_alpine()`으로 분기하고 있었는데, xTCRP와 mshell이 이제 같은 `alpine-redpill` 브랜치를 쓰므로 이 전제 자체가 더 이상 유효하지 않습니다).
- **`bspatch` 실행권한 유실**: `sudo cp`는 원본 권한을 보존하지 않는데, 이 환경의 root umask가 결과물을 `0700`으로 만들어 `tc` 계정에서 보이지 않게 되어 "Some tools weren't available" 빌드 실패로 이어졌습니다.

</details>

## 참고

- 기존 v1.4.1.x 사용자는 직접 업데이트하지 않는 한 영향받지 않습니다.
- 이번 릴리즈의 모든 수정사항은 `_t` 테스트 트랙(`functions_t.sh`/`build-loader_t.sh`)에서 실기(xTCRP + mshell)로 먼저 검증한 뒤 운영에 반영했습니다.
