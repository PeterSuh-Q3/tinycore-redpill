# alpine-redpill v1.4.1.2

빌드 실패 진단 강화, 저사양 스왑 개선, 메뉴 사용성 개선을 담은 릴리즈입니다.

<details>
<summary><b>빌드 실패 시 실제 원인이 화면에 보입니다 (클릭해서 펼치기)</b></summary>

<br>

기존에는 빌드가 실패해도 로그의 마지막 10줄만 보여줬는데, 대부분 "Loader creation failed"류의 정형화된 래퍼 메시지뿐이고 정작 원인(cpio 실패, 다운로드 실패, 메모리 고갈 등)은 화면 밖으로 밀려나 있었습니다. Verbose 모드 설정과 무관하게 항상 이랬습니다.

이제 빌드 실패 시 다음이 **항상**(Verbose 모드 무관) 표시됩니다:
- 로그 전체에서 원인 관련 줄 추출(마지막 10줄만이 아니라)
- cpio/램디스크 리패킹 실패 상세(`/tmp/strerr.log`에는 이미 캡처되고 있었지만 화면에 노출되지 않던 부분)
- 다운로드 실패 진단(HTTP 코드, DNS/연결/TLS 소요시간, curl 자체 에러메시지)
- 실패 시점의 메모리/램디스크 스냅샷: RAM, 스왑, tmpfs 사용량, OOM-killer 발동 여부 — 메모리 문제인지 한 줄 요약으로 즉시 판단 가능

`monitor()`에도 RAM/스왑뿐 아니라 램디스크(tmpfs) 크기/사용량이 함께 표시됩니다.

2.2GB RAM 환경에서 실제 OOM(실기에서는 커널 패닉 `System is deadlocked on memory`까지)을 의도적으로 재현해 검증했습니다.

</details>

<details>
<summary><b>저사양 스왑: zram을 빠른 1차 계층으로 추가 (클릭해서 펼치기)</b></summary>

<br>

저사양(RAM ≤2.1GB) 환경에서 기존 디스크 기반 스왑파일(우선순위 100, 1.5GB) 앞에 zram(우선순위 200)을 추가했습니다. 가벼운 스와핑은 빠른 zram(RAM 내 압축)이 먼저 흡수하고, 정말 부족할 때만 디스크 스왑으로 넘어갑니다. 스왑 지연이 줄고 USB/SD 부팅매체 쓰기 횟수가 줄어드는 효과가 있으며, 물리 RAM을 초과하는 진짜 여유 공간은 여전히 디스크 스왑이 담당합니다(zram 단독으로는 RAM 내 압축일 뿐이라 절대적 상한을 못 늘림).

`cleanupmemory()`도 스왑 장치 하나가 아니라 활성화된 전체를 각자의 우선순위를 유지하며 순회하도록 수정했습니다.

</details>

<details>
<summary><b>그 외 수정 사항 (클릭해서 펼치기)</b></summary>

<br>

- xTCRP(Buildroot friend 커널) 박스가 `/etc/alpine-release` 부재로 인해 관리가 멈춘 `main` 브랜치로 조용히 떨어지던 문제 수정 — 이제 실제 관리 브랜치인 `alpine-redpill`을 사용합니다.
- 메인 메뉴 맨 아래에 "메뉴종료 (exit menu)" 항목 추가 — 기존엔 종료 수단이 전원끄기(Power Off)뿐이었는데, 이제 전원은 그대로 두고 셸로 빠져나올 수 있습니다.
- `tcrp-modules`/`tcrp-addons`/`rp-ext` 브랜치 참조를 존재하지 않는 `master`에서 실제 브랜치인 `main`으로 수정.

</details>

## 참고

- 검증(실기 재테스트)은 이번 릴리즈 배포 이후 별도로 진행합니다.
